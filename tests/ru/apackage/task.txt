Тестовое задание:

Написать  автотесты, используя junit, для объекта (интерфейса) Square
Зафиксировать все найденные проблемы

Пример получения экземпляра объекта находится в классе SquareTest.


Решение:

1. Найдены следующие баги: 

1) Отсутствует спецификация трестируемого объекта (возможно это часть задания).
2) Отстутствует проверка/защита от ввода отрицательных значений длины стороны квадрата.
3) Отстутствует проверка/защита от ввода нулевой длины стороны квадрата.
4) При вводе длины меньшей либо равной 1.0E-17 возвращаемое значение 0.9.
5) При тестировании значений длины от 1.0E-17 до 10.0 (не включая 10.0) получены не корректные возвращаемые значения, например для 1.0 , получено 1.9
(однако, для 10.0 получено корректное значение 10.0).
6) При тестировании значений длины от 10.0 (не включая 10.0) до 20.0 (не включая 20.0) получены не корректные возвращаемые значения, например для 15 , получено 14.9.
(однако, для 20 получено корректное значение 20).
7) При тестировании значений от 20.0 (не включая 20.0) до 30.0 (не включая 30.0) возвращаемые значения - отрицательные, но верные по модулю, например для 25 , получено -25).
 

2. В данной версии обновлен способ ввода тестовых данных, применены “параметризированные аннотации”, также незначительно обновлен список тестовых входных значений.

* SquareTestPrm1 - для проверки возврата длины стороны квадрата (t1)
и для проверки возврата площади квадрата (t2). 
* SquareTestPrm2 - для проверки метода, проверяющего равны ли квадраты по площадям.
